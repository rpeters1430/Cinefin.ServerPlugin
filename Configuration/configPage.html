Docker / reverse-proxy setup?
The plugin runs inside the Jellyfin container. If your apps sit behind a reverse proxy (e.g. https://yourhost.com/sonarr), the Jellyfin container usually cannot reach that external address due to Docker hairpin-NAT restrictions.

Solution: fill in the Internal URL fields below using the Docker container name and port. The plugin will use those for all server-side API calls while the URL above is kept only for display / reference.

Examples for a typical Docker Compose stack:
• Sonarr internal: http://sonarr:8989/sonarr
• Radarr internal: http://radarr:7878/radarr
• Overseerr internal: http://overseerr:5055

Tip: use the Run Diagnostics button at the bottom to test reachability before saving.
